# 前端设计规划

本文档用于规划 Personal Knowledge Agent 前端的阶段性建设。前端开发必须服务于 `AGENTS.md` 中定义的项目定位：构建一个面向个人学习场景的知识管理 Agent 平台，而不是普通聊天页面或展示型官网。

## 技术栈

第一阶段采用以下技术栈：

- React
- TypeScript
- Vite
- Ant Design
- lucide-react
- ECharts 或 Recharts 后续用于评估与学习分析
- SSE 或 WebSocket 后续用于聊天流式响应

前端工程已从静态页面进入真实接口联调阶段。

## 前端产品定位

前端应呈现为一个工作台型应用，强调高效、清晰、可持续使用。界面不做营销型首页，不做大幅装饰性 Hero，而是直接进入可操作的知识管理工作台。

主要用户是正在学习 Java、后端、AI Agent 或准备面试的个人开发者。页面设计应支持反复使用、快速扫描和明确操作。

## 信息架构

第一阶段规划以下一级导航：

- 工作台：展示知识资产、学习进度、最近摄取任务和近期问答。
- 知识库：展示文档、网页文章、标签、摄取状态。
- 采集中心：支持文件上传和网页链接提交的入口设计。
- Agent 问答：面向知识库的 RAG 对话界面。
- 学习计划：展示学习计划、复习卡片和薄弱知识点。
- 评估观测：展示 RAG 检索质量、Token、耗时和引用覆盖率。
- 设置：后续放置模型、知识空间、用户偏好和安全配置。

## 第一阶段目标

第一阶段只完成前端基础框架和关键页面雏形：

1. 创建 Vite + React + TypeScript 前端目录。
2. 创建全局布局，包括侧边栏、顶部栏和内容区域。
3. 创建基础路由状态或页面切换状态。
4. 创建工作台页面，展示项目核心能力入口。
5. 创建知识库页面，展示文档和网页文章的列表雏形。
6. 创建采集中心页面，展示文件上传和 URL 提交的表单样式。
7. 创建 Agent 问答页面，展示问题输入、回答、引用来源和工具调用日志的 UI 雏形。
8. 创建学习计划页面，展示计划、卡片和知识薄弱点。
9. 创建评估观测页面，展示 RAG 指标卡片和调用记录雏形。

第一阶段完成后暂停，等待人工检查和 GitHub 提交。

## 暂不实现内容

以下内容留到后续阶段：

- 不接入真实登录。
- 不接入真实后端 API。
- 不实现真实文件上传。
- 不实现真实网页抓取。
- 不调用真实模型。
- 不实现 SSE 或 WebSocket 流式响应。
- 不引入复杂状态管理库。
- 不实现完整移动端细节。

## 页面设计原则

- 页面直接进入产品工作台，不做营销落地页。
- 功能区域要服务真实使用，不堆叠无意义卡片。
- 导航和核心操作必须清晰。
- 页面文案使用中文，便于项目复盘和简历表达。
- 视觉风格保持克制、专业，适合学习与研发工具。
- 避免单一色系主导，使用中性色搭配少量功能色。
- 所有页面都要为后续 API 接入保留清晰的数据结构。

## 后续阶段建议

第二阶段建议开始接入后端基础接口前的前端准备：

- 抽象 API Client。
- 定义前后端 DTO 类型。
- 添加环境变量配置。
- 增加 loading、empty、error 状态。
- 细化知识库和采集中心页面。

## 第二阶段目标

第二阶段继续保持“不接入真实后端”的边界，只为后续接口接入打基础：

1. 新增前端环境变量示例，明确 API Base URL 配置方式。
2. 新增统一 API Client，封装请求地址、请求头、错误处理和 JSON 解析。
3. 新增前端 DTO 与分页类型，避免页面直接依赖临时 mock 数据结构。
4. 新增统一页面状态组件，用于展示 loading、empty 和 error 状态。
5. 让知识库、采集中心和评估观测页面具备更接近真实业务的状态表达。

第二阶段完成后暂停，等待人工检查和 GitHub 提交。

当前真实后端联调范围：

- 工作台知识、摄取、学习计划和 Agent 调用聚合概览。
- 设置页模型标识、Embedding 标识、引用策略和默认召回数量偏好。
- 知识空间列表。
- 文档列表。
- 摄取任务状态。
- URL 提交任务。
- RAG 问答接口。
- POST SSE 流式增量与引用来源。
- 工具调用摘要与写工具确认弹窗。
- 笔记和复习卡片列表。

工作台已移除 `mockData.ts` 依赖，通过单个聚合接口加载首屏指标、最近知识资产和今日学习任务，
并具备加载状态、空状态、失败提示和手动重试能力。

设置页已从静态配置形态升级为真实后端读写，并使用版本号处理多页面并发保存冲突。

## 第三阶段目标

第三阶段优先完成前端本地运行和联调前准备：

1. 安装前端依赖并生成 `package-lock.json`。
2. 运行 TypeScript 与 Vite 构建检查。
3. 启动 Vite 开发服务器进行页面预览。
4. 补齐设置页，展示 API 地址、模型偏好和知识空间选择。
5. 记录前端运行、构建和预览步骤，形成可重复的验证说明。

如果当前环境无法联网安装 npm 依赖，需要保留代码修改并在文档中记录本地手动验证步骤。

## 第一阶段验收标准

- `ui` 目录具备标准 Vite 前端结构。
- 页面可以通过本地开发服务器启动。
- 导航可以切换主要页面。
- 页面内容体现项目核心目标。
- 代码中保留必要注释，说明 mock 数据和未来 API 接入点。
- 不包含密钥、Token 或机器本地配置。
